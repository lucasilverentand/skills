# Standard pointers and effects
**When possible, utilize the content effects provided by the system.** Users quickly become accustomed to the visual feedback offered throughout the operating system and generally anticipate that this experience will extend to every application they use. To ensure a consistent user journey, your interactions must align with the design purpose of each effect. Specifically:

- Use `highlight` for minor elements that feature a transparent background.
- Use `lift` for minor elements that possess an opaque background.
- Use `hover` for larger components and customize the scale, tint, and shadow properties as necessary (for guidance on this customization, refer to [Customizing pointers](#Customizing-pointers)).
- **Prefer the system's pointer appearances for standard buttons and text input fields.** When the pointer behaves as users expect, you enhance their comfort while using your application.
- **Add padding around interactive elements to establish comfortable hit targets.** You may need to experiment to determine the appropriate size for an element’s hit region. If this region is too small, users may feel compelled to interact with the element with extreme precision. Conversely, if an element’s hit region is excessively large, users might feel the action requires too much effort to move the pointer away from it. Generally, adding approximately 12 points of padding around elements that include a bezel works well; for elements lacking a bezel, adding about 24 points of padding around the visible edges is effective.
- **Ensure contiguous hit regions for custom bar buttons.** If there is a gap between the hit regions of neighboring buttons in a bar, users might experience a distracting movement when the pointer briefly reverts to its default form while transitioning between buttons.
- **Specify the corner radius for a nonstandard element that receives the lift effect.** When using the system-provided `lift` effect, the pointer adapts to match the element’s shape as it fades out. By default, the pointer uses the system-defined corner radius to transition into a rounded rectangle. If your element has a different geometry—such as being a circle—you must provide the radius so that the pointer can animate into the element’s shape seamlessly. For developer guidance, consult [UIPointerShape.roundedRect(_:radius:)](apple:UIKit/UIPointerShape-swift.enum/roundedRect(_:radius:)).
