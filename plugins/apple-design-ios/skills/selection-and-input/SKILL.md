# Selection and input

## When to use
- User asks about **selection and input** on iOS and iPadOS (e.g. `"how do I design selection and input for iPhone"`).
- User is building an iPhone UI that needs selection and input and wants to follow Apple's guidelines.
- User asks to audit or review selection and input in an iOS and iPadOS design.
- User mentions selection and input in the context of an iPhone app, game, or interface.

## Topics
- [Color wells](color-wells.md) — A color well allows users to modify the hue of text, shapes, guides, and other visual elements on screen.
- [Combo boxes](combo-boxes.md) — A combo box integrates a text input area with a drop-down button into a single control.
- [Digit entry views](digit-entry-views.md) — A digit entry view occupies the entire screen, prompting users to input a sequence of numbers (such as a PIN) using a digit-specific keyboard.
- [Image wells](image-wells.md) — An image well is a version of an image view that allows for editing.
- [Pickers](pickers.md) — A picker presents one or more scrollable lists containing distinct values from which users can select.
- [Segmented controls](segmented-controls.md) — A segmented control consists of a linear arrangement of two or more segments, each functioning as a button.
- [Sliders](sliders.md) — A slider is a horizontal track featuring a control, known as a thumb, that users manipulate between defined minimum and maximum values.
- [Steppers](steppers.md) — A stepper is a two-part control used by users to adjust an incremental value up or down.
- [Text fields](text-fields.md) — A text field is a rectangular region where users input or modify short, specific pieces of text.
- [Toggles](toggles.md) — A toggle allows users to select between two opposing states (like on or off), using differing appearances to denote each state.
- [Virtual keyboards](virtual-keyboards.md) — For devices lacking physical keyboards, the system provides different types of virtual keyboards for data entry.
