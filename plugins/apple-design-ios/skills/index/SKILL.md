# Human Interface Guidelines
The Human Interface Guidelines (HIG) offers guidance and established best practices to help you create exceptional experiences on any Apple platform

## When to use
- User asks about **index** on iOS and iPadOS (e.g. `"how do I design index for iPhone"`).
- User is building an iPhone UI that needs index and wants to follow Apple's guidelines.
- User asks to audit or review index in an iOS and iPadOS design.
- User mentions index in the context of an iPhone app, game, or interface.

When designing interfaces for Apple platforms, keep these principles in mind:

##### Hierarchy
Ensure controls and interface elements establish a distinct visual hierarchy, elevating and separating them from the content they govern.

##### Harmony
Maintain a concentric design relationship between the hardware and software to achieve harmony across all interface elements, system experiences, and devices.

##### Consistency
Utilize platform conventions to sustain a cohesive design that dynamically adjusts across different window sizes and displays.
