# Designing object interactions
- **Allow users to manipulate objects directly when feasible.** Direct interaction with onscreen 3D elements is more intuitive and immersive than relying on indirect controls. However, if users are moving around while using the application, indirect controls may be more suitable.
- **Enable direct interaction with virtual objects using standard, recognizable gestures.** For instance, consider supporting a single-finger drag to move items and a two-finger rotation for spinning objects. Refer to **Gestures** for guidance.
- **Generally, maintain interaction simplicity.** Touch gestures are fundamentally two-dimensional, but AR experiences involve the three dimensions of reality. Consider these approaches to simplify user interaction with virtual objects:
- **Respond to gestures near interactive virtual objects.** It can be challenging for users to achieve precision when targeting small, thin, or distant points on objects. When your application detects a gesture close to an interactive object, it is usually best practice to assume the user intends to affect that specific object.
- **Allow users to initiate object scaling if it fits your application's purpose.** For example, if your app allows users to explore a conceptual environment, supporting object scaling is likely appropriate since the app does not need to replicate reality. Conversely, if your app assists shoppers in selecting furniture, allowing them to scale a chair object does not aid visualization of how it will appear in a room.

> **Tip**
> Regardless of the application's goal, do not use scaling to adjust an object's perceived distance. If you enlarge a distant object attempting to make it appear closer, the result is simply a larger object that remains visually far away.

- **Be cautious of potentially conflicting gestures.** For example, a two-finger pinch gesture is similar to a two-finger rotation. If you implement two such comparable gestures, thoroughly test your application to ensure they are interpreted correctly.
- **Ensure virtual object movement aligns with the physics of your app's AR environment.** Users do not necessarily expect an object to move smoothly across a rough or uneven surface, but they do expect objects to remain visible during movement. Aim to keep moving objects anchored to real-world surfaces and prevent them from jumping or disappearing and reappearing when users resize, rotate, or move them.
- **Investigate even more engaging interaction methods.** Gestures are not the only way users can interact with virtual objects in AR. Your app can incorporate other factors, such as motion and proximity, to animate content. For instance, a game character could turn its head toward a person as they approach it.
