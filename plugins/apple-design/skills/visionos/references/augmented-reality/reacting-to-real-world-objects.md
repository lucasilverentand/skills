# Reacting to real-world objects
AR experiences can be enhanced by utilizing known images and objects within the real world to anchor virtual content. For instance, an application that identifies sci-fi film posters could cause virtual spacecraft to emerge from them and navigate the environment. Another use case is an art museum application that triggers a virtual tour guide upon recognizing a specific sculpture. To support such experiences, your app provides ARKit with a collection of 2D reference images or 3D reference objects, and ARKit signals when and where it detects any of these items in the current environment. For detailed developer guidance, refer to [Detecting Images in an AR Experience](apple:ARKit/detecting-images-in-an-ar-experience).

- **When a detected image first disappears, consider delaying the removal of virtual objects attached to it.** Since ARKit does not track changes in the position or orientation of each detected image, waiting up to one second before fading out or removing attached virtual objects helps prevent flickering.
- **Limit the number of reference images in use simultaneously.** Image detection performs optimally when ARKit is searching for 100 or fewer distinct images in the physical environment. If your application requires more than 100 reference images, you should dynamically switch the set of active references based on context. For example, a museum guide app could request location services permission to narrow the search to images displayed in the user's current area.
- **Limit the number of reference images requiring precise positional tracking.** Tracking the position of a reference image consumes additional resources. Use a tracked image only when the image itself is expected to move within the environment, or when the attached animation or virtual object is small relative to the size of the image.
