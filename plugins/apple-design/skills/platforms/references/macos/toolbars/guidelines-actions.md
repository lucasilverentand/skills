# Actions
- **Provide actions that support the main tasks people perform.** Generally, prioritize commands based on how likely users are to want them. While this usually means prioritizing frequently used actions, in certain applications, it may be more appropriate to prioritize commands linked to the most critical or high-level objects users interact with.
- **Make sure the meaning of each control is clear.** Users should not have to guess or experiment to understand what a toolbar item does. Simple, easily recognizable symbols are preferred over text labels for items, unless the action (such as *edit*) cannot be adequately represented by a symbol. For guidance on symbols representing common actions, refer to **Standard icons**. **Prefer system-provided symbols without borders.** System symbols are familiar, automatically handle coloring and vibrancy, and maintain consistent responsiveness to user input. Since the section itself provides a visible container, borders (like those found in outlined circle symbols) are unnecessary because the system automatically manages hover and selection state appearances. See **SF Symbols** for guidance. **Use the `.prominent` style for key actions such as Done or Submit.** This action should be visually distinct and tinted to serve as a clear focal point. Include only one primary action, and place it on the trailing side of the toolbar.
