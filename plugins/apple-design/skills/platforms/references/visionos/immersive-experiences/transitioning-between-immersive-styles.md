# Transitioning between immersive styles
- **Ensure smooth, predictable transitions when altering immersion levels.** Assist users in preparing for different experiences by implementing gentle transitions that allow them to follow visual changes. Avoid abrupt or jarring shifts, as these can cause discomfort or disorientation. For technical guidance, refer to [CoordinateSpaceProtocol](apple:SwiftUI/CoordinateSpaceProtocol).
- **Allow users control over entering or leaving a highly immersive state.** Unexpectedly plunging into a more immersive experience can be disorienting. Instead, provide an explicit action for entering or exiting immersion, allowing users to decide when they wish to be deeply engaged with your content and when they wish to disengage. For instance, Keynote includes a noticeable Exit button within its fully immersive Rehearsal mode to facilitate returning to the slide-viewing window. Do not rely on system controls for users to reduce immersion in your application.
- **Be explicit about the function of any exit control.** Your button must clearly communicate whether it reverts the user to a prior, less immersive context or terminates the entire experience. If exiting your immersive mode also quits the application or game, consider offering controls that allow users to pause or return to a state where they can save their progress before quitting.
