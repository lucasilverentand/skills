# Adaptability
Every application and game must adjust when the device or system context shifts. On iOS, iPadOS, tvOS, and visionOS, the operating system provides a set of *traits* that describe variations in the device environment impacting how your app or game appears. While SwiftUI or Auto Layout can help ensure your interface adapts dynamically to these traits and other contextual shifts, alternative methods must be employed if you do not use those tools.

Here are some of the most common device and system variations that require handling:

- Different screen sizes, resolutions, and color spaces across devices
- Device orientations (portrait/landscape)
- System features such as Dynamic Island and camera controls
- External display support, Display Zoom, and resizable windows on iPad
- Dynamic Type text-size adjustments
- Internationalization features based on locale, including left-to-right/right-to-left layout direction, date/time/number formatting, font variation, and text length
- **Design a layout that adapts gracefully to context changes while remaining recognizably consistent.** Users expect your experience to function correctly and feel familiar whether they rotate their device, resize a window, add another display, or switch devices. You can help maintain an adaptable interface by respecting system-defined safe areas, margins, and guides (when available) and by using layout modifiers to precisely control view placement.
- **Be prepared for text-size changes.** Users appreciate apps and games that respond when they select a different text size. When you support **Supporting Dynamic Type**—a feature allowing users to select the visible text size in iOS, iPadOS, tvOS, visionOS, and watchOS—your app or game can react appropriately to text size adjustments. To implement Dynamic Type in your Unity-based game, utilize Apple’s accessibility plug-in (for developer guidance, refer to **Apple – Accessibility**). For guidance on text display within your application, consult **Typography**.
- **Preview your app on multiple devices, using different orientations, localizations, and text sizes.** You can accelerate the testing phase by initially verifying versions of your experience using both the largest and smallest layouts. Although it is generally recommended to preview features like wide-gamut color on physical devices, the Xcode Simulator can be used to check for clipping and other layout issues. For instance, if your iOS app or game supports landscape mode, the Simulator allows you to confirm layouts appear correct regardless of whether the device rotates left or right.
- **When necessary, scale artwork in response to display changes.** For example, viewing your app or game in a different context—such as on a screen with a varying aspect ratio—might cause your artwork to appear cropped, letterboxed, or pillarboxed. If this occurs, do not alter the artwork's aspect ratio; instead, scale it so that all critical visual content remains visible. In visionOS, the system automatically handles scaling **Scale** a window as it moves along the z-axis.
