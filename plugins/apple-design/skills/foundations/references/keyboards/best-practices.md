# Best practices
**Ensure comprehensive keyboard accessibility when feasible.** Available across iOS, iPadOS, macOS, and visionOS, Full Keyboard Access enables users to navigate and trigger windows, menus, controls, and system functions exclusively via the keyboard. To validate Full Keyboard Access within your application or game, enable it through the Accessibility section of the system-provided Settings app. For developer resources, consult [Support Full Keyboard Access in your iOS app](https://developer.apple.com/videos/play/wwdc2021/10120/) and [isFullKeyboardAccessEnabled](apple:AppKit/NSApplication/isFullKeyboardAccessEnabled).

> **Important**
> While iPadOS supports keyboard navigation within text fields, text views, and sidebars, and provides APIs to support this in collection views and other custom components, refrain from enabling keyboard navigation for controls like buttons, switches, and segmented controls. Instead, allow users to utilize Full Keyboard Access to activate controls, traverse all on-screen elements, and execute gesture-based interactions such as drag and drop. For further information, refer to **iPadOS**; for developer guidance, see [Focus-based navigation](apple:uikit/focus-based_navigation).

**Adhere to conventional keyboard shortcuts.** Most users expect that when using different applications, they can rely on the standard keyboard shortcuts common across the system and other apps. If your application introduces a unique action that users perform often, it is preferable to establish a [Custom keyboard shortcuts](#Custom-keyboard-shortcuts) shortcut for it rather than repurposing a standard combination associated with a different function. In the context of games, users might anticipate certain standard shortcuts—such as Command–Q to exit the game—but they also expect the ability to modify each game’s key bindings to suit their individual play style. For guidance, see **Keyboards**.
