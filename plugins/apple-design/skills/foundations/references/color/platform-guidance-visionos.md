# Platform guidance — visionOS
- **Use color sparingly, especially on glass.** Standard visionOS windows utilize system-defined glass **Materials**, which allows light and objects from the physical environment to be visible through them. Because these environmental colors are visible, they can compromise the legibility of colorful app content within the window. Reserve color usage for situations where it aids in drawing attention to critical information or illustrating relationships between interface components.
- **Prefer using color in bold text and large areas.** Applying color to small or lightweight text can make it difficult for users to perceive and understand.
- **In a fully immersive experience, help people maintain visual comfort by keeping brightness levels balanced.** Although high contrast can effectively direct attention to important content, it may also cause visual discomfort if the user's eyes are accustomed to low light or darkness. Consider making content fully bright only when the rest of the visual context is also brightly lit. For example, avoid displaying a bright object against a very dark or black background, especially if the object flashes or moves.
