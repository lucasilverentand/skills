# Menus and actions

## Topics
- [Activity views](activity-views.md) — An activity view, often referred to as a *share sheet*, presents different tasks available within the current context.
- **Buttons** — A button triggers an immediate action upon interaction.
- **Context menus** — A context menu provides access to functionality directly associated with an item, without cluttering the interface.
- **Dock menus** — On macOS, a secondary click on an application or game icon in the Dock reveals a Dock menu that includes both system-provided and custom items.
- **Edit menus** — An edit menu allows users to make changes to selected content in the current view, while also offering related commands such as Copy, Select, Translate, and Look Up.
- **Home Screen quick actions** — Home Screen quick actions allow users to perform app-specific tasks directly from the Home Screen.
- **Menus** — A menu displays its options when interacted with, offering a space-efficient way to present commands within your app or game.
- **Ornaments** — In visionOS, an ornament presents controls and information related to a window without crowding or obscuring its contents.
- **Pop-up buttons** — A pop-up button displays a menu containing options that are mutually exclusive.
- **Pull-down buttons** — A pull-down button reveals a menu of items or actions that are directly related to the button's intended purpose.
- **The menu bar** — On a Mac or iPad, the menu bar at the top of the screen displays the application's or game's top-level menus.
- **Toolbars** — A toolbar provides convenient access to frequently used commands, controls, navigation features, and search.
