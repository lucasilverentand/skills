# Scribble
With Scribble and Apple Pencil, users can input text wherever it is accepted in your application without needing to tap or switch modes. Since Scribble is natively integrated into iPadOS, it is available across all applications by default.

- **Make text entry feel fluid and effortless.** By default, Scribble functions within all standard text components—including text fields, text views, search inputs, and editable web content—with the exception of password fields. If your app utilizes a custom text field, ensure users do not need to tap or select it before they can begin writing.
- **Make Scribble available everywhere people might want to enter text.** Unlike keyboard input, using Apple Pencil encourages users to treat the screen like paper. To reinforce this perception in your app, make Scribble consistently accessible wherever text entry feels natural. For instance, in Reminders, users naturally create a new reminder by writing in the blank space below the last item, even if that area lacks a dedicated text field. For developer guidance, consult [UIIndirectScribbleInteraction](apple:UIKit/UIIndirectScribbleInteraction-1nfjm).
- **Avoid distracting people while they write.** Certain text field behaviors are suitable for keyboard input but can disrupt the natural writing experience provided by Apple Pencil. For example, it is advisable to prevent displaying autocompletion text while users are writing in a text field because the suggestions may visually interfere with their input. Additionally, hide a field’s placeholder text as soon as users begin writing so that their input does not appear to overlap it.
- **While people are writing in a text field, make sure it remains stationary.** In some scenarios, moving a text field upon gaining focus makes sense (e.g., a search field shifting to accommodate results). This movement is acceptable when using the keyboard, but it can make users feel they have lost control of their input location when writing with Apple Pencil. If you cannot prevent a text field from moving or resizing, consider delaying the change until the user pauses their writing.
- **Prevent autoscrolling text while people are writing and editing in a text field.** When transcribed text scrolls automatically, users may attempt to avoid writing on top of it. Furthermore, if text scrolls while users are using Apple Pencil for selection, they might select a different range than intended.
- **Give people enough space to write.** A small text field can be uncomfortable for handwriting input. If you anticipate Apple Pencil usage, enhance the writing experience in your app by increasing the text field size before users begin or when they pause; avoid resizing a text field while active writing is occurring. For developer guidance, see [UIScribbleInteraction](apple:UIKit/UIScribbleInteraction).
