# Event tickets
Use the event ticket pass style to grant entry into different events such as concerts, movies, plays, and sporting matches. While each pass typically corresponds to a single event, you may also utilize one pass for multiple events, similar to a season ticket.

An event ticket can include logo, strip, background, or thumbnail imagery. However, if you supply a strip image, refrain from including either a background or thumbnail image. You also have the option to include an extra row containing up to four auxiliary fields. For developer guidance, refer to the `row` property within [PassFields.AuxiliaryFields](apple:WalletPasses/PassFields/AuxiliaryFields-data.dictionary).

In iOS 18 and later, the system introduces an advanced style for contactless event tickets called *poster event ticket*. These poster event tickets deliver a rich visual experience, prominently feature the event artwork, provide easy access to supplementary event details, and integrate with system applications like Maps and Weather.

> **Important**
> Poster event tickets are incompatible with tickets that rely on a QR code or barcode for entry.

A poster event ticket displays an event logo and background image, and optionally allows for a separate logo of the ticket issuer or company. The system utilizes metadata about your event to structure ticket information and suggest relevant actions. All poster event tickets require a specific set of metadata defined in [SemanticTags](apple:WalletPasses/SemanticTags), along with an additional required set depending on the event type—general, sports, or live performance. You may also include optional metadata to enhance your ticket further. For instance, you can specify an admission level for a live performance (e.g., General Admission), which the system displays alongside seating details. For developer guidance, consult [Supporting semantic tags in Wallet passes](apple:WalletPasses/supporting-semantic-tags-in-wallet-passes).

The system uses the provided metadata to generate a Maps shortcut for venue directions and an event guide displayed beneath the ticket when viewed in the Wallet app. The event guide offers quick access to information like the weather forecast and venue map, as well as rapid actions such as checking baggage policies or ordering concessions. You can display a minimum of one and up to four quick action buttons in the event guide; if you include more than four, the system groups them into a menu. You may optionally also include supplementary ticket information, such as pre-paid parking details, which the system will display below the ticket.

- **Create a vibrant and engaging background.** Your background image serves as the visual centerpiece of a poster event ticket. Limit text within your artwork, and design an image that is easily recognizable, allowing users to quickly locate their ticket among other passes in their Wallet app. If your background image is a solid color or includes one in the footer, consider setting a matching footer background color to improve integration with the image.
- **Position your background image in the safe area.** The system displays ticket information within the header and footer, which overlap the background image. To prevent content in your artwork from being obscured, ensure it resides within the safe area. For developer guidance, refer to `footerBackgroundColor` in [Pass](apple:WalletPasses/Pass).
- **Ensure sufficient contrast so that ticket information is easy to read.** By default, the system applies a gradient in the header and a blur effect in the footer of your poster event ticket to guarantee adequate contrast between the background image and the ticket information. You may consider adjusting these gradient and blur effects if greater contrast is needed. The system can also automatically determine the optimal text color for ticket information and labels based on your background image. If you choose to customize text colors, ensure the selected color provides sufficient contrast, especially if you define a footer background color or a seat section color to aid wayfinding. For developer guidance, see `useAutomaticColors` in [Pass](apple:WalletPasses/Pass) and `seatSectionColor` in [SemanticTagType.Seat](apple:WalletPasses/SemanticTagType/Seat-data.dictionary). **Consider using the additional information tile for extra event details.** When you possess more event information that users might find useful, the additional information tile below the ticket is an ideal location for it. If you have supplementary details that are crucial to display on the front of the ticket, keep the text concise to prevent cluttering the footer. For developer guidance, see `additionalTicketAttributes` in [SemanticTags](apple:WalletPasses/SemanticTags) and [PassFields.AdditionalInfoFields](apple:WalletPasses/PassFields/AdditionalInfoFields-data.dictionary).
- **Continue to support event tickets for earlier versions of iOS.** Users expect contactless event tickets to function correctly, regardless of their device's operating system version. Continue providing primary, secondary, and auxiliary information within [PassFields](apple:WalletPasses/PassFields) and image assets for your event ticket. This allows the system to automatically generate the appropriate ticket style for a user's device; otherwise, your ticket may appear blank on devices running older versions of iOS.
