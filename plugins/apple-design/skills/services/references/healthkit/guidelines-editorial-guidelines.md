# Editorial guidelines
*   Refer to the application as *Apple Health* or *the Apple Health app*. Using *Apple Health* enhances clarity in your application and marketing copy.

*   Do not employ the term *HealthKit*. This is a developer-facing term identifying the framework your application uses to access health data. If you need to explain how your app interacts with user data, use the term *the Apple Health app*. For instance, you might state that your application “works with the Apple Health app” or “uses data from the Apple Health app.”

*   Ensure correct capitalization when using *Apple Health*. It is two words, with uppercase A and uppercase H, followed by lowercase letters. Display *Apple Health* entirely in uppercase only when required to match an established typographic interface style, such as within an app that capitalizes all text.

*   To prevent user confusion, utilize the system-provided translation for *Health*. It is recommended to refer to the Apple Health app using the term that users see on their device.
