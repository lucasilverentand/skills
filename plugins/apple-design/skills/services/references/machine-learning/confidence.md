# Confidence
Confidence measures how certain a result is. Since not all models generate confidence scores by default, you should consider generating them if they can enhance the user experience of your application.

While it might seem intuitive that higher confidence yields a better result and thus improves the user experience, this is not always true. You must confirm that your confidence values accurately reflect the quality of your results. For instance, you could evaluate values across several confidence thresholds or compare them against multiple versions of your app. If the correlation between your confidence values and result quality is unclear, it is unwise to present confidence information to users.

- **Know what your confidence values mean before you decide how to present them.** For example, users might overlook low-quality results from [critical or complementary](#Critical-or-complementary) features—especially when the results include [attribution](#Attribution) or other contextual details—but prominently displaying poor results is likely to erode trust in your app.
- **In general, translate confidence values into concepts that people already understand.** Simply showing a confidence score does not inherently help users grasp how it relates to the result. For example, a feature suggesting new music based on listening history might calculate a 97% match between a suggested song and the user's usual choices. However, labeling this suggestion with "97% match" provides insufficient information for the user to make a decision. Conversely, providing an attribution that clearly states the underlying behavior—such as "Because you listen to pop music"—is more actionable.
- **In situations where attributions aren’t helpful, consider ranking or ordering the results in a way that implies confidence levels.** If you are required to display confidence directly, consider using semantic categories. For instance, a feature predicting travel prices could replace numerical confidence ranges with terms like "high chance" and "low chance," providing context to the values and aiding user comparison and understanding.
- **In scenarios where people expect statistical or numerical information, display confidence values that help them interpret the results.** Examples include weather forecasts, sports statistics, and polling data, which are often accompanied by specific values expressing the accuracy as an interval or percentage.
- **Whenever possible, help people make decisions by conveying confidence in terms of actionable suggestions.** Understanding user goals is crucial for expressing confidence in a manner that aids decision-making. For example, if your feature predicts the lowest price for an item, users are seeking to optimize their time and spending. For such a feature, providing actionable advice like "This is a good time to buy" or "Consider waiting for a better price" is far more valuable than displaying percentages or other numerical confidence scores.
- **Consider changing how you present results based on different confidence thresholds.** If high or low levels of confidence significantly affect how users experience the results, adapting your presentation is advisable. For instance, when confidence is high, a Photos face recognition feature simply displays photos matching a specific person; conversely, with lower confidence, the feature prompts users to confirm if the photos contain that person before showing more.
- **When you know that confidence values correspond to result quality, you generally want to avoid showing results when confidence is low.** Especially if a feature is proactive and offers unsolicited suggestions, poor results can frustrate users and diminish trust in the feature. For proactive features and suggestions, it is recommended to establish a confidence threshold below which you withhold results.
