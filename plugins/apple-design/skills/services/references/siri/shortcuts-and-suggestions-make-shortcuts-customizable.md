# Make shortcuts customizable
When defining a parameter for each detail your app requires to execute an intent, users can customize the shortcut by editing these details within the Shortcuts application.

To inform users which details they are able to modify and how their changes affect the action, you must supply a *parameter summary*. A parameter summary briefly describes the action by using the parameters in a sentence that begins with a verb. For instance, a customizable *Order coffee* shortcut might display a parameter summary such as “Order *quantity* *coffee*,” where *quantity* and *coffee* are the editable parameters. Here is an example of how the *Order coffee* shortcut might appear after users supply values for the *quantity* and *coffee* parameters.

- **Provide a parameter summary for each custom intent you support.** Your parameter summary must include, at minimum, all parameters your intent requires and any parameters that receive values from other applications or actions. If you omit optional parameters or those unlikely to be edited by users, they can still be accessed in the Show More section.
- **Craft a short parameter summary that’s clearly related to your intent’s title.** When the intent title and the parameter summary are similar, users can easily recognize the action regardless of where they view it. It is beneficial to use identical wording in both the summary and the title, particularly starting both phrases with the same verb. For example, if your intent is titled “Search encyclopedia,” a suitable parameter summary would be “Search encyclopedia for *search term*.”
- **Aim for a parameter summary that reads like a sentence.** Use standard sentence capitalization, but do not include terminal punctuation. If possible, avoid punctuation altogether. Punctuation within a summary—especially colons, semicolons, and parentheses—can hinder readability and comprehension.
- **Provide multiple parameter summaries when necessary.** If your action includes a parameter that has a parent-child relationship with other parameters, you can supply several summary variants based on the current value of the parent parameter. For example, if your *order coffee* shortcut allows users to specify pickup or delivery, your parameter summary should reflect the current choice. In this situation, create one parameter summary to help users select a store location and another summary to assist them in choosing a delivery address. Ensure that all variants maintain consistent grammatical structure and parameter order.
- **Provide output parameters for information that people can use in a multistep shortcut.** For instance, an *order coffee* action might provide output including the estimated delivery time and the order cost. With this information, users could build a multistep shortcut to message a friend about the delivery time or log the transaction in their preferred budgeting application.
- **Consider defining an input parameter.** When you define an input parameter for an action, that action can automatically receive output from a preceding action within a multistep shortcut. For example, if your action applies a filter to the image it receives in an *image* parameter, you might designate *image* as the input parameter so that it automatically accepts images from other actions. You configure an input parameter in your intent definition file (see [Adding User Interactivity with Siri Shortcuts and the Shortcuts App](apple:SiriKit/adding-user-interactivity-with-siri-shortcuts-and-the-shortcuts-app#3239040)).
- **Help people distinguish among different variations of the same action.** For example, an app offering a *send message* action might use a contact photo to help users visually differentiate the different messages they send. To achieve this, select the parameter that is most identifiable to users and designate it as the key parameter (see [Adding User Interactivity with Siri Shortcuts and the Shortcuts App](apple:SiriKit/adding-user-interactivity-with-siri-shortcuts-and-the-shortcuts-app#3239040)). You must provide an image for the key parameter every time you submit the action (for developer guidance, refer to [INImage](apple:Intents/INImage)).
- **Avoid providing multiple actions that perform the same basic task.** For example, instead of offering one action to add text to a note and another action to add an image, consider providing a single action that allows users to include both types of content. Offering a few high-level actions makes it easier for users to understand how the actions combine in a multistep shortcut.

For developer guidance, see [Shortcut-Related UI](apple:SiriKit/shortcut-related-ui).
