# Mistakes
While errors are inevitable in any application, perfection is not expected. However, mistakes can erode user experience and diminish trust. To prevent negative outcomes, it is essential to:

- Anticipate errors. Whenever feasible, design methods to prevent mistakes and mitigate them when they occur.
- Enable users to manage errors. Since mistakes can have varied impacts, the tools provided to handle them must be capable of addressing those consequences.
- Learn from errors if doing so enhances the application. In some instances, deriving lessons from a mistake might have unintended negative effects, such as introducing unpredictability into the user experience. When appropriate, utilize each error as a data point to refine machine learning models and improve the app.

Several machine learning patterns can assist in managing errors:

- [Limitations](#Limitations) help manage user expectations regarding the accuracy of suggestions.
- [Corrections](#Corrections) provide users with a path to success even when the system's output is flawed.
- [Attribution](#Attribution) offers users insight into the source of suggestions, which aids in understanding errors.
- [Confidence](#Confidence) allows you to assess the quality of your results, influencing how they are presented.
- Feedback — both [explicit](#Explicit-feedback) and [implicit](#Implicit-feedback) — allows users to inform you about errors that may have gone unnoticed.
- **Grasp the gravity of a mistake's consequences.** For instance, an incorrect keyboard suggestion might merely annoy users, but suggesting a travel itinerary that leads to a missed flight constitutes a severe inconvenience. Demonstrate empathy by offering corrective actions or tools commensurate with the seriousness of the error.
- **Make it effortless for users to correct frequent or predictable mistakes.** If a simple way to fix errors is not provided, user trust will diminish.
- **Continuously update your feature to reflect users’ evolving interests and preferences and help prevent mistakes.** For example, implicit feedback can be used to detect shifts in user tastes and habits. It is also beneficial to update your feature with domain-specific information, such as current popular trends in entertainment. Ideally, users should not need to perform any actions to benefit from your app's improvements.
- **When possible, address errors without complicating the UI.** Certain patterns, such as corrections and limitations, tend to integrate smoothly into an app's interface, whereas others, like attributions, can be difficult to incorporate. Balance a pattern’s impact on the UI against its potential to amplify the original mistake. For example, if an attribution update is incorrect and displayed in the UI, the effect of the initial error will be magnified.
- **Exercise extreme caution regarding mistakes in proactive features.** A proactive feature—such as a suggestion based on user behavior—promises valuable outcomes without requiring any input from the user. However, because users do not request a proactive feature, they often tolerate its errors less patiently. Mistakes generated by proactive features can also lead users to feel a loss of control.
- **As you work on reducing errors in one area, always consider the effect your work has on other areas and overall accuracy.** For example, optimizing an image-recognition app to better identify dogs might simultaneously decrease its ability to recognize cats. As your models advance, be prepared for errors to evolve as well. Use your understanding of user preferences to guide where you focus your efforts.
