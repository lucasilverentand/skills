# Interface icons
When utilizing **SF Symbols** to provide interface icons for your application, you gain access to variants suited for the RTL context and localized symbols for Arabic, Hebrew, and other languages. If you develop custom symbols, you have the ability to define their directionality. For guidance intended for developers, refer to [Creating custom symbol images for your app](apple:UIKit/creating-custom-symbol-images-for-your-app).

- **Flip interface icons that represent text or reading direction.** For instance, if an icon uses left-aligned bars to signify text in a Left-to-Right (LTR) context, those bars must be right-aligned when used in an RTL context.
- **Consider creating a localized version of an interface icon that displays text.** Some icons incorporate letters or words to convey a script-related concept, such as font size selection or a signature. If your custom interface icon requires displaying actual text, consider developing a localized version. For example, SF Symbols provides distinct versions of the signature, rich-text, and I-beam pointer symbols compatible with Latin, Hebrew, and Arabic scripts, among others.

If your custom interface icon uses letters or words to communicate a concept unrelated to reading or writing, consider designing an alternative image that does not rely on text.

- **Flip an interface icon that indicates forward or backward motion.** When movement occurs in the same direction as reading, users typically interpret this as forward; conversely, they tend to perceive movement in the opposite direction as backward. An interface icon depicting an object moving forward or backward must be flipped in the RTL context to maintain the intended meaning of the motion. For example, an icon representing a speaker usually shows sound waves projecting forward from the speaker. In LTR, these waves originate from the left; therefore, in RTL, the icon must flip to show them originating from the right.
- **Don’t flip logos or universal signs and marks.** Displaying a flipped logo causes confusion and may carry legal risks. Always present a logo in its original orientation, even if it contains text. Users expect universal symbols and marks, such as the checkmark, to maintain a consistent appearance, so avoid flipping them.
- **In general, refrain from flipping interface icons that depict real-world objects.** Unless the object is being used to denote directionality, it is preferable not to flip an icon representing a familiar item. For example, clocks function identically regardless of language direction, so a traditional clock interface icon should appear the same across all languages. While some icons might seem to reference language or reading direction because they represent items slanted for right-handed use, most users are right-handed, making a flip unnecessary and potentially confusing.
- **Before simply flipping a complex custom interface icon, evaluate its individual components and the overall visual balance.** In certain scenarios, a component—such as a badge, slash, or magnifying glass—must adhere to a visual design language irrespective of localization. For example, SF Symbols maintains visual consistency by using the same backslash to signify prohibition or negation of a symbol's meaning in both LTR and RTL versions.

In other instances, you may need to flip a component (or its placement) to ensure the localized version of the icon remains coherent. For example, if a badge represents the actual UI elements visible to users in your app, it must flip if your UI flips. Alternatively, if a badge modifies the meaning of an interface icon, consider whether flipping the badge preserves both its modified meaning and the overall visual equilibrium of the icon.
