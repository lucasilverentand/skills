# Best practices
- **Establish a consistent visual hierarchy that helps communicate the relative importance of different chart elements.** The data itself should typically be the most prominent element, while descriptions and axes provide secondary context without competing with the data.
- **In a compact environment, maximize the width of the plot area to give people enough space to comfortably examine a chart.** To accommodate important data within a limited width, ensure vertical axis labels are as concise as possible while maintaining clarity. You may also consider placing units in another part of the chart, such as a title, and positioning longer axis labels (like category names) inside the plot area if it doesn't obscure critical information.
- **Make every chart in your app accessible.** Charts, like all visual infographics, must be fully accessible to users regardless of how they perceive content. For instance, supporting VoiceOver is essential; it describes on-screen content to help users navigate and gather information without needing visual input (learn more about VoiceOver at [Vision](https://www.apple.com/accessibility/vision/)). Beyond providing accessibility labels for chart components, you can enhance the VoiceOver experience by utilizing Audio Graphs. [Audio graphs](apple:Accessibility/audio-graphs) supply chart data to VoiceOver, which generates a series of tones audibly representing the data values and trends. It also allows you to present high-level text summaries for additional context. Refer to [Enhancing the accessibility of a chart](#Enhancing-the-accessibility-of-a-chart) for guidance.
- **Let people interact with the data when it makes sense, but don’t require interaction to reveal critical information.** In a Stocks application, for example, users are often most interested in performance trends over time. Therefore, the app displays a line graph showing performance for the chosen period (e.g., one day, three months, five years). If users wish to explore finer details, they can drag a vertical indicator across the line graph to reveal the value at that specific time.
- **Make it easy for everyone to interact with a chart.** Sometimes, chart marks are too small to be accurately targeted by a finger or pointer, making the chart difficult for users with motor control limitations and uncomfortable for all users. If this is the case, consider expanding the hit target to encompass the entire plot area, allowing users to scrub across the space to reveal different values.
- **Make an interactive chart easy to navigate when using keyboard commands (including full keyboard access) or Switch Control.** By default, these input methods tend to visit onscreen elements linearly, such as the sequence of values in a data file. If you wish to provide a custom navigation flow for your chart, two primary approaches are available. The first is using accessibility APIs (like [accessibilityRespondsToUserInteraction(_:)](apple:SwiftUI/View/accessibilityRespondsToUserInteraction(_:))) to define a logical and predictable path through the chart's information. For example, you might allow navigation along the X axis rather than jumping between points. The second method—particularly useful for very large datasets—is allowing users to shift focus among subsets of values instead of navigating every individual data point. Note that both customization options can also improve the VoiceOver experience, even if your chart is not interactive. See **Accessibility** for detailed guidance.
- **Help people notice important changes in a chart.** If users fail to detect shifts in marks or axes, they risk misinterpreting the data. Animating these changes can help draw attention to them, but you must also highlight the changes through other means to ensure that VoiceOver users and those who disable animations are aware of them. For developer instructions, consult [UIAccessibility.Notification](apple:UIKit/UIAccessibility/Notification) (UIKit) or [NSAccessibility.Notification](apple:AppKit/NSAccessibility-swift.struct/Notification) (AppKit).
- **Align a chart with surrounding interface elements.** For instance, aligning the leading edge of a chart with the leading edge of other views on the screen often works well. One way to maintain a clean leading edge is by displaying the label for each vertical grid line on its trailing side. You might also consider shifting the Y axis to the trailing edge of the chart so that its tick labels do not extend past the chart's leading edge. If a label appears unanchored, you can use a tick to tie it to a grid line.
