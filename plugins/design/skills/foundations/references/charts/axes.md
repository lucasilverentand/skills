# Axes
**Select a fixed or dynamic axis range based on the chart's intended meaning.** In a *fixed* range, the upper and lower limits of the axis remain constant. Conversely, in a *dynamic* range, these bounds adjust according to the displayed data. Employ a fixed range when specific minimum and maximum values hold significance across all potential datasets. For instance, users anticipate a chart displaying battery charge to have a minimum of 0% (empty) and a maximum of 100% (full).

Conversely, use a dynamic range when the possible data values fluctuate widely and the goal is to utilize the entire plotting area. For example, the Y axis range in the Health app’s Steps chart varies so that the highest number of steps recorded for a particular time period approaches the top of the visualization.

- **Determine the lower bound value according to the mark type and how the chart is used.** For example, bar charts function effectively when zero is used for the Y axis lower bound, as this allows users to visually compare relative bar heights and estimate values reasonably well. In contrast, setting a lower bound of zero can sometimes complicate the discernment of meaningful differences between values. For instance, a heart rate chart that always uses zero as its lower bound might obscure important variations between resting and active readings if those differences occur in a range far from zero.
- **Favor common numerical sequences in the axis tick and grid-line labels.** For example, if a standard number sequence like 0, 5, 10 is used, users are likely to instantly understand that each tick value exceeds the previous one by five. Although a sequence like 1, 6, 11 follows the same rule, its lack of commonality often requires users to spend extra time calculating the interval between values.
- **Adjust the appearance of grid lines and labels to match the chart's specific use case.** Too many grid lines can overwhelm the visual field, distracting users from the data; conversely, too few may make it difficult to estimate a mark’s value. To help determine the appropriate density and visual weight of these elements, consider the chart's context within the interface, the supported interactions, and the tasks users perform with the visualization. For example, if users can inspect individual data points by interacting with a chart, you might use fewer grid lines and lighter label colors to ensure the data remains visually prominent.
