# Best practices
- **Design icons that are instantly recognizable and highly simplified.** Excessive detail can render an interface icon confusing or illegible. Aim for a straightforward, universal design that users will quickly grasp. Generally, icons function best when they employ familiar visual metaphors directly linked to the actions they trigger or the content they represent.
- **Ensure visual uniformity across all interface icons within your application.** Whether you exclusively use custom icons or combine them with system-provided ones, all interface icons in your app must maintain consistency in size, level of detail, stroke thickness (or weight), and perspective. Depending on the icon's visual weight, you may need to adjust its dimensions to guarantee it appears consistent alongside other icons.
- **Generally, match the weights of interface icons and adjacent text.** Unless you intend to disproportionately emphasize either the icons or the accompanying text, using matching weights provides a cohesive appearance and consistent level of importance for your content.
- **If necessary, add padding to a custom interface icon to achieve optical alignment.** Certain icons—particularly asymmetric ones—may appear unbalanced when geometrically centered rather than optically centered. For instance, the download icon shown below carries more visual weight in its lower half than its upper half, which can make it appear too low if geometrically centered.

In such scenarios, you can slightly shift the icon's position until it achieves optical centering. When creating an asset that includes these adjustments as padding around the interface icon (as shown below on the right), you can optically center the icon by geometrically centering the entire asset.

Optical centering adjustments are typically minor, but they can significantly affect your app's visual presentation.

- **Provide a selected-state version of an interface icon only when required.** You do not need to supply both selected and unselected appearances for an icon used within standard system components like tab bars, toolbars, or buttons. The operating system automatically manages the visual representation of the selected state.
- **Use inclusive imagery.** Consider how your icons can be understandable and welcoming to all users. Favor gender-neutral human depictions and avoid imagery that may be difficult to interpret across diverse cultures or languages. For guidance, consult **Inclusion**.
- **Include text in your design only when it is critical for conveying meaning.** For example, using a character within an interface icon to denote text formatting is often the most direct way to communicate that concept. If you must display individual characters in your icon, ensure they are localized. If suggesting a passage of text, design an abstract representation and include a mirrored version for use in right-to-left contexts. For guidance, see **Right to left**.
- **If you create a custom interface icon, use a vector format such as PDF or SVG.** The system automatically scales a vector-based interface icon for high-resolution displays, eliminating the need to provide multiple versions. Conversely, PNG—used for app icons and other images requiring effects like shading, textures, or highlights—does not support scaling, necessitating multiple versions for each PNG-based interface icon. Alternatively, you can create a custom SF Symbol and specify a scale that aligns the symbol's emphasis with adjacent text. For guidance, see **SF Symbols**.
- **Provide alternative text labels for custom interface icons.** Alternative text labels, or accessibility descriptions, are not visible but allow VoiceOver to audibly describe the on-screen content, simplifying navigation for users with visual impairments. For guidance, see **VoiceOver**.
- **Avoid replicating Apple hardware products.** Hardware designs are prone to frequent changes and can quickly make your interface icons or other content appear outdated. If you must display Apple hardware, use only the images found in [Apple Design Resources](https://developer.apple.com/design/resources/) or the SF Symbols that represent different Apple products.
