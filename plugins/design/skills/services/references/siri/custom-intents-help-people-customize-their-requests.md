# Help people customize their requests
Custom intents allow for follow-up questions, enabling users to refine a single intent's outcome and achieve more. For instance, if you implement an *order coffee* intent, follow-up questions like “What size?”, “What flavor?”, and “Which location?” guide users to their desired outcome. Details such as size, flavor, and location are defined by your app as *parameters* to help users customize their request.

Users provide parameter values through follow-up responses or by modifying existing choices within the Shortcuts app. For example, if an *order ground coffee* intent includes a grind size parameter, you might prompt with, “Which grind?” If the typical order is coarse ground, you can streamline the interaction by setting *coarse* as the default parameter value in a dialogue such as, “Do you want coarse-ground coffee?” If they select a different grind, you can prompt again with the full range of options. In voice-only scenarios, Siri presents your follow-up questions and relays the responses to you. When users edit a parameter value using the Shortcuts app, you receive the updated value upon execution of the associated shortcut. For technical guidance, refer to [Adding User Interactivity with Siri Shortcuts and the Shortcuts App](apple:SiriKit/adding-user-interactivity-with-siri-shortcuts-and-the-shortcuts-app).

- **Design intents that require as few follow-up questions as possible.** Often, an intent can fulfill a request without requiring any subsequent questions. While follow-up prompts increase flexibility, forcing users into an extended interaction is undesirable. In most scenarios, limiting the interaction to one or two follow-up questions is optimal.
- **List the smallest number of options possible, and sort the items in a way that makes sense.** Similar to excessive follow-up questions, presenting too many choices can make task completion feel burdensome. When deciding whether to include an option, weigh its utility against its complexity. For example, in a food ordering application, users may find it easier to process a list of individual menu items than a list of orders that contain multiple components. After narrowing down to a small set of useful options, consider sorting them by popularity, frequency, or how recently they were used.
- **Make sure each follow-up question is meaningful.** Ideally, every follow-up question assists the user in making a critical selection. If the options or questions presented are overly granular or too similar, the conversation risks becoming repetitive, leading users to perceive your intent as too much effort.
- **Design parameters that are easy for people to understand and use.** Aim for parameters that represent simple attributes or values, naming them using clear, straightforward terminology. For instance, a soup ordering application might define parameters for the soup type, serving size, and delivery location using simple names like *soup*, *size*, and *location*. Refer to [Shortcuts and suggestions](#Shortcuts-and-suggestions) for guidance.
- **Ask for confirmation only when necessary.** An intent may require confirmation before completing a task or upon interpreting an answer to a follow-up question. Apps handling tasks with financial implications, such as placing orders, must obtain confirmation before finalizing the transaction. For non-financial tasks, requiring confirmation can feel like unnecessary overhead and may discourage users from completing their request. In all cases, limit requests for confirmation to once.
- **Support follow-up questions when it makes sense.** For example, a food ordering application might offer pickup or delivery options but should only request a specific location after the user has selected the delivery method.
- **Prioritize the options you offer based on the context in which people run your shortcut.** For instance, if users utilize your shortcut to place a pickup order, present nearby pickup locations. Offering options that adapt to the execution context helps users avoid needing separate shortcuts for specific scenarios.
- **Consider adjusting the parameter values you offer when people set up your shortcut.** When indicating that a parameter has dynamic options, you can improve the shortcut setup experience in two ways:
- You can identify and present parameter values relevant to the user's current context during shortcut setup. For example, if users select a value for a store-location parameter using the Shortcuts app, that parameter can dynamically generate a list of stores closest to the device.
- You can provide an exhaustive list of parameter values. When setting up a shortcut, having a comprehensive range of options assists users in creating the desired shortcut. Conversely, when using a shortcut to accelerate an action, users generally prefer a shorter selection list for convenience.

For developer guidance, see the `storeLocation` parameter within the intent definition file of the [Soup Chef: Accelerating App Interactions with Shortcuts](apple:SiriKit/soup-chef-accelerating-app-interactions-with-shortcuts) sample code project.
