# Notification actions
A notification may include a configurable detail view that allows users to execute actions using up to four buttons without launching your application. For instance, a Calendar event notification might feature a Snooze button to delay the alarm for several minutes. For guidance on implementation, refer to [Handling notifications and notification-related actions](apple:UserNotifications/handling-notifications-and-notification-related-actions).

- **Ensure the actions offered are valuable and relevant to the notification's context.** Prioritize actions that enable users to complete common, time-saving tasks without needing to enter your app. For each button, use a concise, title-cased phrase or term that clearly communicates the outcome. Do not include your application's name or any extra details in the button label; keep the text brief to prevent truncation, and consider localization during composition.
- **Do not offer an action whose sole purpose is to launch your app.** Since users expect related content when they tap a notification or its preview, including an action button that performs this same function clutters the detail view and can cause confusion.
- **Favor non-destructive actions.** If a destructive action is necessary, ensure users have sufficient context to prevent accidental execution. The system provides a unique visual indicator for actions you designate as destructive.
- **Supply a simple, recognizable interface icon for every notification action.** An interface icon reinforces the meaning of an action, allowing users to instantly grasp its function. The system displays your interface icon alongside the action title on the trailing side. When utilizing [SF Symbols](sf-symbols.md), you can select an existing symbol representing your command or modify a related symbol to generate a custom icon.
