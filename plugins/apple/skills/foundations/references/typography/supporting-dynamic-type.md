# Supporting Dynamic Type
Dynamic Type is a system-level feature across iOS, iPadOS, tvOS, visionOS, and watchOS that allows users to modify the displayed text size on their device for optimal comfort and readability. For related information, consult **Accessibility**.

For a list of available Dynamic Type sizes, refer to [Specifications](#Specifications). You may also download Dynamic Type size tables in the [Apple Design Resources](https://developer.apple.com/design/resources/) for each platform.

For developer guidance, see [Text input and output](apple:SwiftUI/Text-input-and-output). To support Dynamic Type in Unity-based games, utilize [Apple’s Unity plug-ins](https://github.com/apple/unityplugins).

- **Ensure your application's layout accommodates all font sizes.** Confirm that your design scales appropriately, and that text and glyphs remain legible across the full range of font sizes. On iPhone or iPad, activate Larger Accessibility Text Sizes via Settings > Accessibility > Display & Text Size > Larger Text and verify your app remains comfortably readable.
- **Scale the size of important interface icons as font sizes increase.** If you use interface icons to convey critical information, ensure they remain easily viewable even when the font size is large. When using **SF Symbols**, you benefit from icons that automatically scale with Dynamic Type size adjustments.
- **Minimize text truncation as font sizes increase.** Generally, strive to display as much essential text at the largest accessibility font size as you do at the largest standard font size. Avoid truncating text within scrollable areas unless users can access a separate view to read the remainder of the content. You can prevent text truncation in a label by configuring it to utilize as many lines as necessary to display a useful amount of text. For developer guidance, see [numberOfLines](apple:UIKit/UILabel/numberOfLines).
- **Consider adjusting your layout when using large font sizes.** When the font size increases within a horizontally constrained environment, inline elements (such as glyphs and timestamps) and container boundaries can compress text, leading to truncation or overlapping. To enhance readability, consider employing a stacked layout where text appears above secondary items. Multicolumn text may also become less readable at large sizes due to horizontal space limitations. Reduce the number of columns when the font size increases to prevent truncation and improve readability. For developer guidance, see [isAccessibilityCategory](apple:UIKit/UIContentSizeCategory/isAccessibilityCategory).
- **Maintain a consistent information hierarchy regardless of the current font size.** For instance, keep primary elements positioned toward the top of a view, even when the font size is very large, so users do not lose track of these elements.
