# Best practices
- **Support the default system-defined menus and their ordering.** Users anticipate finding menu items arranged in a familiar sequence. Often, the system provides standard functionality for these menu selections, reducing your implementation burden. For instance, when text is selected in a typical text field, the system automatically enables the Edit > Copy command.
- **Always show the same set of menu items.** Maintaining visibility of menu options assists users in understanding all actions your application supports, even if those actions are currently unavailable within the context. If a menu bar item cannot be executed, disable the action rather than removing it from the menu structure.
- **Represent menu item actions with familiar icons.** Icons help users quickly identify common functions across your application. Use the same system icons to represent actions like Copy, Share, and Delete, regardless of where they appear. For a collection of icons representing common actions, consult [Standard icons](icons.md#Standard-icons). For further direction, refer to [Menus](menus.md).
- **Support the keyboard shortcuts defined for the standard menu items you include.** Users expect to utilize the keyboard combinations they are accustomed to for common commands such as Copy, Cut, Paste, Save, and Print. Only define custom keyboard shortcuts when absolutely necessary. Guidance is available at [Standard keyboard shortcuts](keyboards.md#Standard-keyboard-shortcuts).
- **Prefer short, one-word menu titles.** different elements—including display size variations and the presence of extra menu bar items—can impact the spacing and visual presentation of your menus. One-word titles are particularly effective in the menu bar because they conserve space and allow for quick scanning. Should a title require more than one word, employ title-style capitalization.
