# Offering interactivity
- **Ensure that tapping the Live Activity launches your application precisely where needed.** Direct users immediately to relevant actions and details; do not require them to navigate extensively to find information. For developer guidance on SwiftUI views that enable deep linking to specific screens, consult [Linking to specific app scenes from your widget or Live Activity](apple:WidgetKit/Linking-to-specific-app-scenes-from-your-widget-or-Live Activity).
- **Prioritize simple and straightforward actions.** Interactive elements like buttons or toggles consume screen real estate that could otherwise display valuable information. Only include interactive components for core functionality directly tied to your Live Activity, especially if users activate it once or pause/resume temporarily (e.g., music playback, workouts, or apps using the microphone for live audio recording). If interactivity is provided, limit it to a single control to minimize accidental taps.
- **Allow users to respond to event or progress updates.** If a Live Activity update requires user input, consider providing a button or toggle for them to act upon it. For instance, a rideshare app's Live Activity might feature a button allowing users to contact the driver while awaiting pickup.
