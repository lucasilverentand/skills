# Platform guidance — macOS
- **Allow users to drag content from your application into the Finder.** If you implement this feature, ensure the content is presented in a format that your application can subsequently open. For instance, Calendar allows users to drag an event into the Finder as a `.ics` file. This file can then be shared or dragged back into Calendar to view it. When necessary, you may output dragged content within a *clipping*, which serves as a temporary storage container for the dragged data. For example, most system applications allow users to drag text to the Finder, where it appears as a clipping. Later, this clipping can be dragged into a text field or another location that accepts text. Note that a drag-and-drop clipping is distinct from the Clipboard.
- **Permit users to drag selected content from a window that is not currently active without first bringing that window into focus.** Selected content in an inactive window is referred to as a *background selection* and visually differs from content selected in the active window. Generally, users expect to drag a background selection into the active window without causing the inactive window to become foregrounded.
- **When feasible, allow users to drag single items from an inactive window without affecting any existing background selection.** For example, a user can drag an unselected file from an inactive Finder window without causing any of the window's currently selected files to become deselected.
- **Consider displaying a badge during drag operations involving multiple items.** A badge is a small, filled oval containing a number that indicates how many items the user is dragging. If the destination can only accept a subset of the dragged items, update the badge to reflect this reduced number.
- **Consider changing the cursor appearance to indicate the outcome of dropping content.** In addition to utilizing the *copy* pointer, you may wish to employ the *drag link*, *disappearing item*, or *operation not allowed* pointers, depending on the specific scenario. For detailed guidance, refer to [Pointers](pointing-devices.md#Pointers).
- **To the greatest extent possible, allow users to select and drag content in a single motion.** Unless the user is selecting multiple items, they appreciate not having to pause between making the selection and initiating the drag operation.
