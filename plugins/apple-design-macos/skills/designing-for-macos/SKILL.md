# Designing for macOS
Macs are relied upon for demanding tasks—including deep productivity work, media consumption, and gaming—due to their power, spaciousness, and adaptability, often requiring multiple applications simultaneously

## When to use

- User is building a Mac UI that needs designing for macos and wants to follow Apple's guidelines.
- User asks to audit or review designing for macos in a macOS design.
- User mentions designing for macos in the context of a Mac app, game, or interface.

When beginning the design of an application or game for macOS, it is essential to first understand the core device characteristics and interaction patterns that define the macOS experience. Basing your design choices on these attributes will help you create an app or game that Mac users value.

- **Display.** Macs typically feature a large, high-resolution screen, and users often extend their workspace by connecting external displays, including an iPad.
- **Ergonomics.** Users generally operate a Mac while seated and stationary, usually placing the device on a desk or table. In typical usage scenarios, viewing distance ranges from approximately one to three feet.
- **Inputs.** Users expect to control the interface and enter data using any combination of input methods, such as physical [Keyboards](keyboards.md), [Pointing devices](pointing-devices.md), [Game controls](game-controls.md), and [Siri](siri.md).
- **App interactions.** Interactions can range from brief, quick tasks lasting only a few minutes to extended periods of deep concentration spanning several hours. Users frequently maintain multiple applications open and expect seamless transitions between active and inactive states when switching contexts.
- **System features.** macOS offers several built-in functionalities that allow users to interact with the system and their applications in ways that are consistent and familiar.
- [The menu bar](the-menu-bar.md)
- [File management](file-management.md)
- [Going full screen](going-full-screen.md)
- [Dock menus](dock-menus.md)

### Best practices
Exceptional Mac experiences integrate the platform and device capabilities that users value most. To ensure your design feels native to macOS, prioritize incorporating these features and capabilities:

- Utilize large displays to present greater amounts of content with fewer nested layers and less reliance on modality, while maintaining an information density that allows users to view content without strain.
- Allow users to control the size, visibility, and placement of your windows to match their workflow and device setup, and support full-screen mode for an uninterrupted experience.
- Employ the menu bar to provide users with readily accessible commands within your application.
- Enable high-precision input modes, allowing users to perform selections and edits with pixel accuracy.
- Implement keyboard shortcuts to accelerate actions and support a keyboard-driven workflow.
- Support personalization, allowing users to customize toolbars, configure windows to display frequently used views, and select preferred interface colors and fonts.
