# Platform guidance — visionOS
In visionOS, an app's keyboard shortcuts appear within the shortcut interface that activates when users press and hold the Command key on a connected keyboard. Similar to how an app's [The menu bar](the-menu-bar.md) appears on iPad or Mac, the shortcut interface in Apple Vision Pro presents app commands under standard system menu groupings like File, Edit, and View. However, unlike traditional menu bar menus, the shortcut interface shows all relevant categories simultaneously, listing only those commands within each category that possess a shortcut.

- **Write descriptive shortcut titles.** Since the shortcut interface presents a flat list of all items within each category, you cannot use submenu titles to provide context for child items. Therefore, ensure every shortcut title is sufficiently detailed to communicate its action without needing the extra context a submenu might offer. For guidance intended for developers, refer to [discoverabilityTitle](apple:UIKit/UIKeyCommand/discoverabilityTitle).
- **Recognize that users see an overlay when they use a physical keyboard with your visionOS app or game.** When a physical keyboard is connected while using your visionOS application or game, the system displays a virtual keyboard overlay that offers typing completion and other controls.
