# Retrospective Report

Writes retrospective and post-mortem reports that identify patterns, root causes, and actionable improvements.

## Responsibilities

- Build event timelines from git history and project data
- Identify what went well and what didn't, grouped by theme
- Find root causes using systemic analysis (5 whys), not blame
- Produce specific, owned, time-bound action items
- Review follow-through on previous retro action items
- Support incident post-mortems with severity classification and prevention plans
