# Hono Skills

Skills for building web applications with [Hono](https://hono.dev) - a fast, lightweight web framework for the edge.

## Skills

| Skill | Description |
|-------|-------------|
| [init](./init/) | Initialize new Hono projects with runtime and template selection |
| [best-practices](./best-practices/) | Hono application architecture and patterns |
| [middleware](./middleware/) | Create and use Hono middleware |
| [auth](./auth/) | Authentication with better-auth, JWT, OAuth, and sessions |
| [validation](./validation/) | Request validation with Zod, Valibot, or custom validators |
| [api-design](./api-design/) | API design patterns including REST, RPC, and OpenAPI |
| [testing](./testing/) | Testing Hono applications |
| [debugging](./debugging/) | Debug and troubleshoot Hono applications |
| [deployment](./deployment/) | Deploy to Cloudflare, Vercel, AWS, and other platforms |
| [database](./database/) | Database integrations with Drizzle, Prisma, D1, and Turso |
| [jsx](./jsx/) | Server-side rendering with Hono JSX |
| [websockets](./websockets/) | WebSocket implementation for real-time features |

## Usage

```
/hono-init                     # Create new Hono project
/hono-auth jwt                 # Implement JWT authentication
/hono-deployment cloudflare    # Deploy to Cloudflare Workers
```

## Supported Runtimes

- Cloudflare Workers
- Deno
- Bun
- Node.js
- Vercel Edge
- AWS Lambda
