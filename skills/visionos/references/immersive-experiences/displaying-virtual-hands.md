# Displaying virtual hands
When an immersive application or game enters a Full Space, it has the option to request permission to conceal a user's physical hands and display virtual representations instead.

- **Prefer virtual hands that match familiar characteristics.** For instance, synchronize the positions and movements of the viewer's hands to allow them to interact with your application or game naturally. When virtual hands behave realistically, they help users maintain immersion within completely virtual environments.
- **Use caution if you create virtual hands that are larger than the viewer’s hands.** Virtual hands substantially exceeding human scale can obstruct the content users wish to view and lead to awkward interactions. Furthermore, oversized virtual hands may appear disproportionate to the environment, suggesting they are unnaturally close to the viewer's face.
- **If there’s an interruption in hand-tracking data, fade out virtual hands and reveal the viewer’s own hands.** Do not allow the virtual hands to appear unresponsive or stuck. Once hand-tracking data resumes, fade the virtual hands back into view.
