# Helping people place objects
- **Indicate when a surface has been located and an object can be placed.** You may utilize the system's coaching view to assist users in identifying a flat, horizontal or vertical surface where an object should be positioned. Once ARKit confirms a surface, your application can display a custom visual cue indicating placement feasibility. Aligning this indicator with the detected surface plane helps users understand how the placed object will appear in their environment.
- **Upon object placement, integrate it into the AR scene without delay.** Although surface detection improves accuracy over time, it is preferable not to wait for perfect data before placement. Use the available information to provide an immediate response when a user places an object; subsequent refinement of the object's position can occur once surface detection is complete. For instance, if an object is placed outside the detected bounds, gently adjust it back onto the surface. Refer to [ARTrackedRaycast](apple:ARKit/ARTrackedRaycast) for guidance on refining object positions.
- **Consider assisting users in locating virtual objects that are off-screen.** Finding an object positioned outside the camera view can be challenging for users. In such scenarios, you can aid discovery by providing auditory or visual cues. For example, if an object is off-screen to the left, displaying a guide along the screen's left edge encourages users to pan the camera in that direction.
- **Do not attempt to achieve precise alignment of objects with detected surface edges.** In AR, surface boundaries are approximations that may shift as the environment undergoes further analysis.
- **Utilize plane classification data to govern object placement.** For example, you might restrict users to placing a virtual piece of furniture only on a plane classified as "floor," or require a "table" classification to place a virtual game board.
