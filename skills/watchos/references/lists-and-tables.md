# Lists and tables
Tables and lists display information arranged across one or more columns of rows

## Platform guidance — watchOS
When feasible, restrict the number of displayed rows. While brief lists are easier for users to review, some users may anticipate a comprehensive list of items. For instance, if a user subscribes to numerous podcasts, they might assume an error exists if they cannot see every item. To make extensive lists more navigable, display the most critical items and include a mechanism for viewing additional content.

If you wish to enable vertical page-based navigation, keep the detail views constrained in length. Users utilize this vertical swiping mechanism to move between different list row details. This method accelerates the experience because users do not have to return to the main list to select a new detail item; however, it is only effective when the detail views are brief. If your detail views require scrolling, users will be unable to employ vertical page-based swiping.
