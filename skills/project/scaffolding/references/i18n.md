# i18n

Internationalization across web (Astro/React) and native (Expo) apps. Translation files live in a shared workspace package.

## Setup

1. Create `packages/i18n/`
2. Structure: `src/locales/en.json`, `src/types.ts`, `src/utils.ts`, `src/index.ts`

### Type-safe keys

```ts
import type en from "./locales/en.json";

type NestedKeys<T, Prefix extends string = ""> = T extends object
  ? { [K in keyof T]: K extends string
      ? NestedKeys<T[K], Prefix extends "" ? K : `${Prefix}.${K}`>
      : never }[keyof T]
  : Prefix;

export type TranslationKey = NestedKeys<typeof en>;
```

### Translation function

```ts
export function t(translations: Record<string, unknown>, key: TranslationKey, params?: Record<string, string | number>): string {
  const parts = key.split(".");
  let value: unknown = translations;
  for (const part of parts) value = (value as Record<string, unknown>)?.[part];
  if (typeof value !== "string") return key;
  if (!params) return value;
  return value.replace(/\{\{(\w+)\}\}/g, (_, k) => String(params[k] ?? `{{${k}}}`));
}
```

## Adding locales

1. Copy `en.json` to `<locale>.json`, translate all values
2. Export from `src/index.ts`
3. Run `tools/translation-check.ts` to verify all keys are present

## Locale detection

- **Web**: URL path prefix, then cookie, then Accept-Language header
- **Expo**: `getLocales()` from `expo-localization`

## Pluralization

Use ICU-style plural rules with `_zero`, `_one`, `_other` suffixes and `Intl.PluralRules`.

## RTL support

```ts
const RTL_LOCALES = ["ar", "he", "fa", "ur"];
export function isRTL(locale: string): boolean { return RTL_LOCALES.includes(locale); }
```

- Web: `<html dir={isRTL(locale) ? "rtl" : "ltr"}`
- Expo: `I18nManager.forceRTL(isRTL(locale))` (requires app restart)
- Use `start`/`end` instead of `left`/`right` in styles
