# Platform guidance — iOS & iPadOS — overview
iPadOS builds upon the established pointer interaction model, dynamically adjusting the pointer to fit the current context and offering detailed visual feedback. This precision enhances productivity and streamlines typical touchscreen operations. The iPadOS pointing system offers users an alternative method for engaging with applications and content; it supplements, rather than replaces, touch input.

- **Allow multiple selection in custom views when necessary.** Starting with iPadOS 15 and newer versions, users can click and drag the pointer across several items to select them. As this interaction occurs, it transforms into a visible rectangle indicating the selected range. Standard nonlist collection views inherently support this multiple selection; however, if your custom views require it, you must implement the functionality yourself. For development guidance, consult [UIBandSelectionInteraction](apple:UIKit/UIBandSelectionInteraction).
- **Distinguish between pointer and finger input only if it provides value.** For instance, a scrubber bar allows users to target a specific location in video playback using the pointer. In this scenario, users can manipulate the playhead via either touch or pointer, but they gain the advantage of using the pointer to select a highly precise seek destination.
