# Layout and organization

## Topics
- **Boxes** — A box defines a visually distinct grouping for logically related components and information.
- **Collections** — A collection manages an ordered sequence of content, presenting it through a highly visual and customizable layout.
- **Column views** — A column view, also referred to as a *browser*, enables users to navigate and view data hierarchies using multiple vertical columns.
- [Disclosure controls](disclosure-controls.md) — Disclosure controls allow users to show or hide information and functionality associated with specific views or controls.
- **Labels** — A label is static text that users can read and often copy, but which cannot be modified.
- **Lists and tables** — Lists and tables structure data into rows and one or more columns.
- [Lockups](lockups.md) — Lockups integrate several distinct views into a single, interactive unit.
- [Outline views](outline-views.md) — An outline view displays hierarchical data within a scrolling list of cells arranged into columns and rows.
- **Split views** — A split view handles the display of multiple adjacent content panes, each capable of holding different components such as tables, collections, images, and custom views.
- **Tab views** — A tab view displays multiple mutually exclusive content panes within a single area, allowing users to switch between them using a tabbed control.
