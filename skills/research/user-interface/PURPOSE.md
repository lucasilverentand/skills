# User Interface

Design system research, component patterns, visual design, and responsive patterns.

## Responsibilities

- Research design systems and component patterns
- Evaluate visual design approaches
- Define responsive design patterns
- Audit component consistency and token usage across the codebase
- Catalog icon, color, and typography usage for design system alignment
- Analyze breakpoint coverage and layout behavior across screen sizes

## Tools

- `tools/token-audit.ts` — scan stylesheets and Tailwind classes for design token usage and deviations
- `tools/component-inventory.ts` — crawl the component directory and generate a categorized component catalog
- `tools/breakpoint-report.ts` — analyze responsive breakpoint usage across all layout files and flag gaps
