# Providing feedback
Drag and drop is a dynamic process that can lead to different outcomes. To ensure users maintain control during this process, providing continuous and clear feedback is essential.

- **Display a drag image once the user has dragged a selection after approximately three points.** It is effective to use a translucent representation of the content being moved. Translucency helps differentiate this representation from the original content and allows users to see potential drop targets as they pass over them. Keep this drag image visible until the content is dropped.
- **If it enhances clarity, modify the drag image to help users predict the outcome of the drag-and-drop operation.** For instance, if a photo is dragged into a document, the drag image could enlarge to reflect the default size it will occupy in the document. You may also utilize drag *flocking* to visually group multiple dragged items—allowing users confirmation that they have included all intended items—and then ungroup them upon dropping. Although changing the drag image offers valuable feedback, avoid making the experience distracting by allowing constant or radical changes to the drag image.
- **Indicate whether a destination can accept dragged content.** For example, you might display an insertion point or highlight a container view only when the destination is valid for accepting the dragged item, and provide no visual feedback—or an explicit "not allowed" indicator, such as the `circle.slash` from SF Symbols—when it cannot accept the item. Display highlighting or other visual cues exclusively while the content is positioned above the destination, removing this feedback when the user moves the dragged content away. If there are multiple potential destinations, provide visual cues that help users focus on one target at a time.
- **Provide visual feedback when an item is dropped onto an invalid destination or when the drop fails.** For example, the item could return to its original location (if the source remains visible), or it could scale up and fade out, suggesting evaporation rather than a successful placement.
